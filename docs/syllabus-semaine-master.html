Code module M1ARCH - Architecture des Réseaux avec Cloud et Modèle OSI
Niveau Master 1 - Unité 1 - Semaine 3
Durée 5 jours / 35 heures + modules optionnels (10h)
Prérequis Fondamentaux réseaux, TCP/IP, administration système Linux/Windows

2 PRÉREQUIS DÉTAILLÉS

Acquis requis des semaines précédentes du cycle M1

SEM1 - M1PISC (Piscine DevOps Container) :

  • Maîtrise de l'administration système Linux (Ubuntu/CentOS)
  • Compétences Docker : création, gestion et orchestration de containers
  • Bases DevOps : intégration continue, pipelines CI/CD
  • Ligne de commande avancée et scripting Bash
  • Virtualisation et concepts d'infrastructure as code

SEM2 - M1INTR (Introduction Cybersécurité et E-Reputation + Gestion Incidents)

  • Fondamentaux de la cybersécurité et analyse de risques
  • Compréhension des menaces réseau et vulnérabilités courantes
  • Protocoles de sécurité de base (HTTPS, SSH)
  • Gestion d'incidents et procédures de réponse
  • Notions de monitoring et détection d'intrusions

Compétences techniques générales requises

  • Réseaux : Modèle TCP/IP, adressage IPv4/IPv6, subnetting
  • Système : Administration Windows Server et Linux
  • Sécurité : Concepts de base, pare-feu, authentification
  • Cloud : Notions d'infrastructure cloud (IaaS, PaaS, SaaS)
  • Outils : Familiarité avec les outils de diagnostic réseau

3 PROFIL DU FORMATEUR

  • Architecture réseau : Expertise en conception et déploiement d'infrastructures réseau
  • Cloud computing : Maîtrise des plateformes AWS, Azure, GCP
  • Protocoles de routage : Configuration avancée RIP, OSPF, BGP
  • VPN et sécurité : Expertise IPSec, OpenVPN, PKI
  • Migration cloud : Expérience architectures hybrides et SD-WAN
  • Certifications recommandées : CCNA/CCNP, Azure/AWS Network Specialty

4 OBJECTIFS

À la fin de ce module, l'étudiant sera capable de :

  • Maîtriser l'architecture du modèle OSI et les protocoles avancés
  • Configurer et administrer les services réseau (DHCP, DNS, HTTP)
  • Implémenter des solutions de load balancing et optimisation HTTP
  • Analyser et sécuriser les communications réseau dans un environnement professionnel
  • Concevoir des architectures réseau haute disponibilité avec intégration cloud

5 PROGRAMME DÉTAILLÉ

Jour 1 - Fondamentaux et Modèle OSI

Contenu Format
Rappels TCP avancés : persistance, pipelining, flow control, congestion control Cours
Modèle OSI : couches 1-4 avec analyse approfondie des PDU Cours + TP
Configuration et administration serveur DHCP (Windows/Linux) TP

Jour 2 - Protocole HTTP avancé

Contenu Format
PDU HTTP : GET, POST, PUT, DELETE, HEAD, TRACE - analyse détaillée Cours + TP
En-têtes HTTP et codes de status (1xx à 5xx) - implémentation pratique TP
Cookies : attributs, options, sécurité et gestion de session TP

Jour 3 - HTTP avancé et Optimisation

Contenu Format
Authentification HTTP : Basic, Improved Digest, Bearer Token Cours
Redirection, hôte virtuel, proxy cache et tunneling HTTP TP
QCM d'évaluation intermédiaire Évaluation

Jour 4 - Load Balancing et Architecture Cloud

Contenu Format
Accélération HTTP, proxy et Web Load Balancing (algorithmes L4/L7) Cours + TP
Architecture réseau cloud : VPC, Subnets, Security Groups TP
Content Delivery Networks (CDN) et optimisation de performances Projet

Jour 5 - Sécurité et Projet Final

Contenu Format
Sécurité réseau : Firewalls, WAF, DDoS Protection Cours + TP
Architectures hybrides et monitoring réseau avancé TP
Projet final : architecture réseau haute disponibilité avec load balancing Projet

6 ENVIRONNEMENT TECHNIQUE

  • Serveurs réseau : Windows Server, Linux (Ubuntu/CentOS) pour DHCP/DNS
  • Équipements réseau : Routeurs Cisco (GNS3/Packet Tracer) pour modules optionnels
  • Outils réseau : Wireshark, nmap, traceroute, netstat, curl avancé
  • Load Balancers : HAProxy, Nginx, Apache HTTP Server
  • VPN/Sécurité : OpenVPN, StrongSwan (IPSec), certificats X.509
  • Monitoring : PRTG, Nagios, ou solutions cloud (CloudWatch)
  • Virtualisation : VirtualBox/VMware pour labs réseau complexes
  • Proxy/Cache : Squid, Varnish pour tests de performance HTTP

9 MODULES OPTIONNELS AVANCÉS

Protocoles de routage avancés

Module optionnel : 3 heures supplémentaires

  • RIP (Routing Information Protocol) : Configuration, métriques, limitations
  • OSPF (Open Shortest Path First) : Areas, LSA, algorithme Dijkstra
  • BGP (Border Gateway Protocol) : AS, peering, route policies
  • Comparaison des protocoles et choix architecturaux

VPN et sécurité réseau avancée

Module optionnel : 4 heures supplémentaires

  • IPSec : AH, ESP, modes tunnel/transport, IKE
  • VPN SSL/TLS : OpenVPN, configuration et déploiement
  • Site-to-Site vs Remote Access VPN
  • Authentification et certificats X.509
  • Performance et optimisation des tunnels VPN

Transition réseaux classiques vers cloud hybride

Module optionnel : 3 heures supplémentaires

  • Migration progressive : Stratégies lift-and-shift vs cloud-native
  • Connectivité hybride : Direct Connect (AWS), ExpressRoute (Azure)
  • SD-WAN : Software-Defined WAN pour l'interconnexion cloud
  • Gestion de la latence et optimisation inter-sites
  • Étude de cas : migration d'architecture d'entreprise
